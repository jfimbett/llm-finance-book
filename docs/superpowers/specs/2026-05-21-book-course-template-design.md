# Book-Course Template — Design Spec
**Date:** 2026-05-21
**Status:** Approved

---

## 1. Purpose

A reusable, AI-assisted template for creating:
- A structured **course** (lecture notes, slides, exercises)
- A companion **LaTeX book** (deeper treatment: proofs, references, appendices)
- **Companion code** (Jupyter notebooks for demos + a shared importable Python package)

Designed for both academic and industry practitioners. All AI instructions are plain markdown files — portable across Claude Code, Cursor, Copilot, ChatGPT, or any LLM. Claude Code's `.claude/` system is the primary runtime, but nothing is locked to it.

---

## 2. Key Conventions

- **Numbering alignment:** Chapter N and Lecture N cover the same topic. Book goes deeper; course is a subset.
- **TOPIC.md** is the single source of truth for project configuration. It is written by `/interview-me` and read by all hooks and agents. Never edit manually after initialization.
- **docs/quality/** accumulates JSON score reports per file. `docs/STATUS.md` is auto-generated from these reports.
- **LaTeX toolchain:** pdflatex + BibTeX/Biber.
- **Code:** Python. Notebooks import from `code/src/` package.

## 4. Agents

All agents live in `.claude/agents/`. Each file defines: persona statement, scope (what to check), output format, and explicit out-of-scope items.

### 4.1 Content Production

| File | Role |
|------|------|
| `book-writer.md` | Drafts LaTeX chapter content: prose, theorems, proofs, examples |
| `lecture-writer.md` | Drafts lecture notes and Beamer slides derived from the chapter |
| `exercise-designer.md` | Creates exercises with difficulty tags (beginner/intermediate/advanced) and model solutions |
| `figure-designer.md` | Writes TikZ code or matplotlib scripts for diagrams and plots |
| `code-writer.md` | Writes companion notebooks and `src/` package code |

### 4.2 Research & Verification

| File | Role |
|------|------|
| `literature-reviewer.md` | Searches for relevant sources, suggests citations, flags missing prior work |
| `math-checker.md` | Steps through derivations line by line, flags proof gaps, checks notation consistency |
| `statistics-reviewer.md` | Reviews statistical methodology, checks assumptions, tests, and interpretations |
| `fact-checker.md` | Verifies factual claims, flags outdated references or superseded results |
| `code-reviewer.md` | Reviews notebooks and scripts for correctness, reproducibility, and style |

### 4.3 Editorial

| File | Role |
|------|------|
| `editor.md` | Rewrites for clarity, flow, and concision without changing technical content |
| `humanizer.md` | Removes AI-generated formulaic phrasing; makes text sound natural |
| `proofreader.md` | Grammar, spelling, punctuation, LaTeX formatting errors |
| `pedagogy-reviewer.md` | Checks learning objectives, scaffolding, and difficulty progression across lectures |
| `accessibility-reviewer.md` | Flags unexplained jargon, assumed background, or unclear explanations |

### 4.4 Consistency & Architecture

| File | Role |
|------|------|
| `consistency-checker.md` | Audits notation, terminology, and symbol use across all chapters |
| `cross-ref-checker.md` | Verifies all `\ref`, `\cite`, `\label` references resolve; checks lecture↔chapter alignment |
| `structure-reviewer.md` | Reviews overall narrative arc, chapter ordering, and coverage gaps |

### 4.5 Scoring

| File | Role |
|------|------|
| `scorer.md` | Scores content on 5 dimensions (clarity, rigor, completeness, pedagogy, style) 0–10 each; outputs JSON to `docs/quality/` |
| `critic.md` | Gives blunt structured criticism with ranked issues (blocker / major / minor); does not soften |
| `peer-reviewer.md` | Simulates anonymous peer review; outputs accept / minor-revision / major-revision / reject + line comments |

## 5. Skills

All skills live in `.claude/skills/`. Each file defines: trigger, step-by-step instructions, inputs, expected outputs, and failure handling.

### 5.1 Creation Workflows

| File | What it does |
|------|-------------|
| `new-topic.md` | Scaffolds a new chapter + lecture + notebook unit with numbered folders, placeholder files, and a TODO.md |
| `draft-chapter.md` | Pipeline: load TOPIC.md → literature-reviewer → book-writer → math-checker → editor |
| `draft-lecture.md` | Load chapter draft → lecture-writer → pedagogy-reviewer → editor |
| `draft-exercises.md` | Load lecture → exercise-designer → math-checker → solutions sign-off |
| `draft-figures.md` | Load chapter section → figure-designer → TikZ/matplotlib output |
| `draft-code.md` | Load lecture + chapter → code-writer → code-reviewer → sync with notebooks |

### 5.2 Quality & Refinement Workflows

| File | What it does |
|------|-------------|
| `score-content.md` | Invoke scorer on a target file; write JSON report to `docs/quality/`; print summary |
| `critique.md` | Invoke critic; produce ranked issue list with severity |
| `peer-review.md` | Invoke peer-reviewer; produce structured review with verdict |
| `refine-until-threshold.md` | **Iteration loop:** score → if any dimension < threshold → invoke relevant agent to fix → re-score → repeat until pass or max iterations reached |
| `full-review.md` | Run critic + peer-reviewer + scorer in sequence; consolidate into one report |

### 5.3 Consistency & Integrity Workflows

| File | What it does |
|------|-------------|
| `audit-notation.md` | Run consistency-checker across all chapters; output symbol table; flag conflicts |
| `audit-cross-refs.md` | Run cross-ref-checker; list broken `\ref`/`\cite`/`\label`; flag misaligned lecture↔chapter numbers |
| `sync-lecture-chapter.md` | Compare a lecture and its paired chapter; list gaps in either direction |
| `audit-bibliography.md` | Check `.bib` for duplicates, missing fields, uncited entries |

### 5.4 Book Production Workflows

| File | What it does |
|------|-------------|
| `build-book.md` | Run pdflatex → biber → pdflatex × 2; report errors |
| `build-slides.md` | Compile all Beamer `.tex` files in `course/lectures/` |
| `release-chapter.md` | Gate: score ≥ threshold + no broken refs + proofreader pass → commit with structured message |

### 5.5 Project Management Workflows

| File | What it does |
|------|-------------|
| `commit-progress.md` | Stage changed files → generate structured commit message (type/scope/summary) → commit |
| `session-summary.md` | List what changed, score deltas, and open items at session end |
| `topic-status.md` | Print status table for every chapter/lecture: draft / in-review / passing / released |

### 5.6 Configuration

| File | What it does |
|------|-------------|
| `interview-me.md` | **Entry point for the template.** Interactive interview that configures the entire project. See Section 7. |

## 6. Hooks

Hooks live in `.claude/hooks/` (Claude Code) with portable copies in `hooks/`. All hooks are shell scripts.

### 6.1 Integrity Hooks (pre-commit — block bad state)

| Script | Trigger | What it does |
|--------|---------|-------------|
| `check-latex.sh` | pre-commit | `pdflatex -draftmode` on `book/main.tex`; blocks if compilation fails |
| `check-refs.sh` | pre-commit | Scans `\ref{}`/`\label{}` mismatches; blocks on undefined references |
| `check-bib.sh` | pre-commit | Validates `.bib` entries have required fields; warns on uncited entries |
| `check-notation.sh` | pre-commit | Diffs symbol table against last known-good version; warns on new undefined symbols |
| `check-numbering.sh` | pre-commit | Verifies chapter N and lecture N exist as a pair; warns if counterpart is missing |

### 6.2 Automation Hooks (reduce manual work)

| Script | Trigger | What it does |
|--------|---------|-------------|
| `auto-commit.sh` | post-tool-use (file write) | Stages changed file; creates structured commit `feat(ch01): draft introduction section` |
| `update-status.sh` | post-commit | Reads `docs/quality/` reports; regenerates `docs/STATUS.md` |
| `session-log.sh` | stop (session end) | Appends one-line summary to `docs/SESSION_LOG.md`: date, files changed, score deltas |
| `scaffold-pairs.sh` | post-tool-use (new file) | When `book/chapters/NN-*/` is created, auto-creates matching `course/lectures/NN-*/` and `code/notebooks/NN-*/` skeletons |

### 6.3 Iteration Hooks (drive the quality loop)

| Script | Trigger | What it does |
|--------|---------|-------------|
| `score-on-save.sh` | post-tool-use (`.tex` or `.md` write) | Runs `score-content` skill on saved file; writes report to `docs/quality/`; prints pass/fail |
| `gate-check.sh` | pre-commit | Reads quality report for files being committed; blocks if any dimension < threshold from TOPIC.md |
| `iterate.sh` | triggered by `gate-check.sh` on failure | Invokes `refine-until-threshold` skill on failing file; re-runs `gate-check.sh`; exits after N iterations |
| `notify-threshold.sh` | post-commit | When a file first passes all dimensions, prints "chapter released" banner; logs to `docs/STATUS.md` |

## 7. /interview-me Skill — Project Configuration Entry Point

When a user clones the template, the first command is `/interview-me`. The skill drives an interactive session, one question at a time, mixing multiple-choice and free-text input.

### Interview Questions

| # | Question | Type |
|---|----------|------|
| 1 | What is your course/book about? | Free text |
| 2 | Who is the primary audience? | Multiple choice: academic / industry / both |
| 3 | How many chapters/lectures are planned? | Free text |
| 4 | Does the book go deeper than the course, or are they 1-to-1? | Multiple choice |
| 5 | What programming language for companion code? | Multiple choice: Python / R / Julia / none |
| 6 | Minimum quality score to pass (per dimension, 0–10) | Multiple choice: 6 / 7 / 8 / 9 |
| 7 | Maximum refinement iterations before human review required | Multiple choice: 3 / 5 / 10 / unlimited |
| 8 | Auto-commit after every AI file write? | Multiple choice: yes / no |
| 9 | Git remote URL (GitHub/GitLab) | Free text or skip |
| 10 | Author name, institution, email | Free text |
| 11 | License | Multiple choice: CC-BY / CC-BY-NC / MIT / All rights reserved |

### Outputs

- Writes all answers to `TOPIC.md` (YAML frontmatter + human-readable summary)
- Fills `book/preamble.tex` author, title, and institution fields
- Creates `.gitignore` appropriate for LaTeX + Python
- Initializes git repo and makes the first commit if not already initialized
- Prints a "ready" summary with the three recommended next commands:
  - `/new-topic` — scaffold your first chapter
  - `/draft-chapter` — start writing
  - `/topic-status` — check overall progress

### TOPIC.md Schema

```yaml
---
title: ""
author: ""
institution: ""
email: ""
license: ""
audience: ""          # academic | industry | both
language: ""          # python | r | julia | none
chapters_planned: 0
book_depth: loose     # loose | coupled
quality_threshold: 7
max_refine_iterations: 5
auto_commit: true
git_remote: ""
---
```

## 8. Quality Loop Design

The quality loop is the core feedback mechanism. It runs automatically via hooks and can also be triggered manually via the `refine-until-threshold` skill.

```
User saves .tex or .md
        ↓
score-on-save.sh → scorer agent → JSON report in docs/quality/
        ↓
All dimensions ≥ threshold?
   YES → green; file is commit-eligible
   NO  → iterate.sh fires
        ↓
iterate.sh → refine-until-threshold skill
   → identifies lowest-scoring dimension
   → maps dimension to agent: clarity→editor, rigor→math-checker, completeness→structure-reviewer, pedagogy→pedagogy-reviewer, style→humanizer+proofreader
   → invokes that agent on the file
   → agent rewrites/fixes the file
   → re-score
        ↓
Loop until pass OR max_refine_iterations hit
   HIT LIMIT → flag for human review; block auto-commit
```

Score dimensions: **clarity** · **rigor** · **completeness** · **pedagogy** · **style**

Each score is 0–10. All must reach `quality_threshold` (from TOPIC.md) before a file passes.

## 9. Open Questions / Future Scope

- Slide format: Beamer assumed, but PowerPoint/reveal.js export could be added later.
- Multi-language support: template is English-first; LaTeX preamble supports UTF-8 but non-Latin scripts would need LuaLaTeX.
- Student-facing repo variant: a separate skill `prepare-student-repo.md` could strip solutions and unreleased chapters before publishing.
