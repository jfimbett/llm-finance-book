01

Recap — the formulas you will need

Differential privacy as a budget, and how to measure leakage after the fact.
Privacy as noise

To hide one person, blur the answer just enough

Differential privacy adds calibrated noise so that no single client's record visibly changes the model's output. Tighter privacy means more noise.

  • Smaller \(\varepsilon\) (the privacy "spend") → more noise → stronger privacy, weaker accuracy
  • Clipping each example's gradient caps how much one record can push the model
Membership-inference AUC
The after-the-fact leakage check: 0.5 = no leakage, 1.0 = total failure; <0.6 commonly acceptable.
The Gaussian mechanism sets the noise scale from the sensitivity \(\Delta_2 f\): \[ \sigma = \frac{\Delta_2 f \cdot \sqrt{2\ln(1.25/\delta)}}{\varepsilon}, \qquad \mathcal{M}(D) = f(D) + \mathcal{N}(0,\sigma^2 I). \] Privacy spent over \(k\) mechanism applications grows by advanced composition: \[ \varepsilon' \approx \varepsilon\sqrt{2k\ln(1/\delta')}. \] DP-SGD clips each per-example gradient to bound sensitivity to \(C\): \[ \hat{g}_i = g_i / \max\!\big(1,\ \|g_i\|_2 / C\big). \]
De-identification toolkit

Two layers find the secrets; three strategies hide them

Two-layer NER pipeline
  • Learned model — PERSON, ORG, LOCATION, MONEY, DATE
  • Rules / regex — IBAN, ISIN, LEI, tax ID, SWIFT
Three strategies
  • Masking — [PERSON_001]
  • Generalisation — 75008 → Paris
  • Suppression — remove the span
\(k\)-anonymity
Output could plausibly come from \(\geq k\) individuals. Masking gives large \(k\) — but context can still re-identify.
pseudonymisation ≠ anonymisation Pseudonymisation keeps text natural but the mapping is a key. GDPR Recital 26 ⇒ still protected personal data, not anonymised.

Problem 1 · 12 min

Can the bank afford four private fine-tunes a year?

A bank fine-tunes a 7B model on credit-decision narratives with DP-SGD. Gradients clipped to \(C = 1.0\); target per run \(\varepsilon_{\text{run}} = 0.8\), \(\delta = 10^{-5}\).

  1. Task A. The clipped gradient has \(\ell_2\)-sensitivity \(\Delta_2 f = C = 1.0\). Compute the noise scale \(\sigma\) for one mechanism application. Use \(\ln(1.25/10^{-5}) \approx 11.7\).
  2. Task B. The team plans \(k = 4\) independent runs over the year. Using advanced composition with \(\delta' = 10^{-5}\) (\(\ln(1/\delta') \approx 11.5\)), what is the total budget \(\varepsilon'\)?
  3. Task C. The annual ceiling is \(\varepsilon_{\max} = 5\). Are the four runs within budget? How many runs fit before the ceiling is breached?

03

Case study — the membership-inference audit

The empirical check the chapter insists you run after every fine-tune.
Case study · the question

How much does a fine-tuned model leak about its training set?

A model that memorised a document fits it suspiciously well. An attacker exploits exactly that gap to guess who was in the training data.

The procedure
  1. Hold out a random subset of documents as the non-member (negative) class.
  2. Score every member and non-member with the likelihood-ratio attack (target vs. reference model).
  3. Sweep the threshold; plot the ROC; report the AUC.

Notebook: code/notebooks/15-privacy-local-models/exercises.ipynb
Figure generator: gen_dp_privacy_utility.py (privacy–utility curve).

acceptance gate AUC ≈ 0.5 ⇒ attack no better than chance (good). AUC > 0.6 ⇒ leakage: add DP-SGD, reduce epochs, or exclude the most sensitive docs.
Case study · the code (1/2)

The attack: does the target fit this doc better than a reference?

If the fine-tuned model assigns a document a much lower loss than an untouched reference model, that document was probably in its training set.

import numpy as np
from sklearn.metrics import roc_auc_score

# Per-document loss under target (fine-tuned) and reference models.
# Lower target loss on a doc => evidence it was a training member.
def membership_scores(target_loss, ref_loss):
    # Likelihood-ratio signal: high when target fits doc better than ref.
    return ref_loss - target_loss          # larger => more "member-like"

def audit_auc(target_loss, ref_loss, is_member):
    scores = membership_scores(target_loss, ref_loss)
    return roc_auc_score(is_member, scores)
Case study · the code (2/2)

A toy run where members overfit, non-members don't

# Toy: members fit better under target model.
rng = np.random.default_rng(0)
ref     = rng.normal(2.0, 0.4, size=400)
member  = ref[:200] - rng.normal(0.6, 0.2, size=200)
nonmemb = ref[200:] - rng.normal(0.0, 0.2, size=200)
target  = np.concatenate([member, nonmemb])
is_member = np.array([1]*200 + [0]*200)

auc = audit_auc(target, ref, is_member)
print(f"MI AUC = {auc:.3f}")
# <0.6 OK; ~0.5 ideal; 1.0 = severe leakage.
Case study · diagnostic questions

Run it, then connect over-fitting to leakage

  1. Q1. What AUC does the toy run report? Acceptable privacy? Which array (member vs. nonmemb) drives the gap, and why?
  2. Q2. Re-run member with the over-fit offset reduced from 0.6 to 0.0. What happens to the AUC — and what does that say about over-fitting and leakage?
  3. Q3. In gen_dp_privacy_utility.py, the curve is \(\text{acc}(\varepsilon)=0.90-(0.90-0.55)e^{-0.45\varepsilon}\). Compute accuracy at \(\varepsilon=1\) and \(\varepsilon=10\). How large is the gap to the non-private baseline (0.90)?
  4. Q4 (link the two). You must keep MI-AUC < 0.6 and accuracy > 0.80. Is a single DP fine-tune at \(\varepsilon=1\) likely to satisfy both? Which lever (clip norm \(C\), epochs, LoRA) would you pull first?

Solutions — the worked answers

Problem 1 budget arithmetic; Problem 2 mask and re-identification.
Solution · Problem 1

One private run fits the budget — not four

The headline: with these settings the bank can afford a single DP fine-tune before the annual ceiling is breached. Privacy is a scarce, depleting resource.

  • A — noise scale: \(\sigma \approx 6.05\)
  • B — four runs: \(\varepsilon' \approx 7.67\)
  • C — vs. ceiling 5: \(7.67 > 5\) → over budget; only 1 run fits
the lesson Budget is scarce — lower \(\varepsilon_{\text{run}}\) per run or fine-tune less often.
Task A (\(\Delta_2 f = 1\), \(\varepsilon_{\text{run}}=0.8\), \(\ln(1.25/10^{-5})\approx 11.7\)): \[ \sigma = \frac{1 \cdot \sqrt{2 \times 11.7}}{0.8} = \frac{\sqrt{23.4}}{0.8} \approx \frac{4.84}{0.8} \approx 6.05. \] Task B (\(k=4\), \(\varepsilon=0.8\), \(\ln(1/\delta')\approx 11.5\)): \[ \varepsilon' \approx 0.8\sqrt{2\times4\times11.5} = 0.8\sqrt{92} \approx 0.8 \times 9.59 \approx 7.67. \] Task C (\(\varepsilon_{\max}=5\)): largest \(k\) with \(0.8\sqrt{2k\cdot 11.5}\le 5\): \[ \sqrt{23k}\le 6.25 \Rightarrow 23k \le 39.1 \Rightarrow k \le 1.7 \;\Rightarrow\; \textbf{1 run}. \]

Appendix · stretch A

Gradient inversion: sharing updates can still leak data

Federated learning shares only gradient updates, never raw data — yet gradient inversion can reconstruct inputs from a single update.

  1. A1. Explain why a FedAvg update \(\Delta_k^t\) from a client training on one sensitive batch can leak that batch — and why larger local batch size and more local steps \(E\) both reduce the leakage.
  2. A2. You add local DP-SGD before transmitting \(\Delta_k^t\). State the two new costs this imposes (relative to plain FedAvg) and why they are unavoidable.
  3. A3. A cross-border bank runs one federated client per jurisdiction. Which regulation does this specifically satisfy that pure centralised training would violate, and why does federation alone not discharge GDPR Art. 17?
Appendix · stretch B

Quantisation audit: sign off a 4-bit on-prem model

A risk team must approve a 4-bit (AWQ) quantised, DP-fine-tuned model for on-prem deployment.

  1. B1. The team worries quantisation might change the privacy guarantee. Argue, using the post-processing immunity property of DP, why quantising a DP-trained model does not weaken its \((\varepsilon,\delta)\) guarantee.
  2. B2. AWQ needs a calibration set. Why is using a small public corpus (not client data) both sufficient for quality and necessary for privacy?
  3. B3. 7B at 4-bit ≈ 3.5 GB; a branch workstation has a single 8 GB GPU. Estimate whether a 13B 4-bit model (≈ 6.5 GB) also fits, and name one operational reason you might still prefer the 7B at the branch.
