01

From sequences to attention

Why reading text in order is hard for a machine — and how attention let models look anywhere at once.
First problem: documents are long

How do you turn a 100-page filing into one vector?

Earnings calls run about 10,000 words; 10-K filings about 100,000. Three ways to compress all those word-vectors into a single document representation — each with a catch.

MethodIdeaWeakness
Mean embeddingAverage every word's vectorNo word order
TF-IDF weightedAverage, but weight rare/distinctive words moreStill bag-of-words
SBERT (contextual)A BERT fine-tuned to embed whole sentencesCompute cost at encoding
Mean pooling and TF-IDF pooling of word embeddings \(\varphi(w_i)\): \[ \bar{\varphi}(d) = \frac{1}{n}\sum_{i=1}^n \varphi(w_i), \qquad \varphi_{\text{tfidf}}(d) = \frac{\sum_i \text{tfidf}(w_i)\,\varphi(w_i)}{\sum_i \text{tfidf}(w_i)}. \] SBERT instead uses a Siamese BERT fine-tuned on NLI pairs to produce a sentence embedding directly.
why order matters Under mean pooling, "Revenue rose, costs fell" \(\equiv\) "Costs rose, revenue fell" — identical vectors, opposite meaning. SBERT advantage: pre-compute embeddings once, then search is \(O(1)\) at query time.
Reading in order · the first neural attempt

RNNs read word by word — and forget the beginning

A recurrent network passes a running "memory" forward as it reads each word. The flaw: over a long document that memory signal fades exponentially, so the network can't connect something on page 3 to something on page 20.

Vanilla RNN recurrence and its backprop-through-time gradient: \[ \mathbf{h}_t = \sigma\!\left(W_h\,\mathbf{h}_{t-1} + W_x\,\mathbf{x}_t + \mathbf{b}\right), \qquad \frac{\partial \mathcal{L}}{\partial \mathbf{h}_t} = \sum_{s=t}^{T} \frac{\partial \ell_s}{\partial \mathbf{h}_s} \prod_{k=t}^{s-1} \frac{\partial \mathbf{h}_{k+1}}{\partial \mathbf{h}_k}. \] Proposition (vanishing & exploding gradients). With \(\rho = \|W_h\|_2\) and \(M_\sigma = \sup_z |\sigma'(z)|\), \[ \Big\|\prod_{k=t}^{T-1}\tfrac{\partial \mathbf{h}_{k+1}}{\partial \mathbf{h}_k}\Big\|_2 \;\leq\; (M_\sigma\,\rho)^{T-t}. \] For the logistic sigmoid \(M_\sigma = \tfrac14\); if \(M_\sigma\rho < 1\) the gradient vanishes exponentially in length.
the finance implication A 10,000-token earnings call makes long-range dependencies — a guidance number on page 3 tied to a caveat on page 20 — practically unlearnable for a plain RNN.
The first fix · LSTM

LSTMs add a memory "conveyor belt" that doesn't fade

The LSTM keeps a separate cell state that information can ride along almost untouched, with learned gates deciding what to forget, what to add, and what to output.

  • Forget gate — what to drop from memory
  • Input gate — what new information to write
  • Output gate — what to expose as the hidden state
\[ \begin{aligned} \mathbf{f}_t &= \sigma(W_f[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_f) &&\text{forget gate}\\ \mathbf{i}_t &= \sigma(W_i[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_i) &&\text{input gate}\\ \mathbf{g}_t &= \tanh(W_g[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_g) &&\text{candidate cell}\\ \mathbf{c}_t &= \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \mathbf{g}_t &&\text{cell state (additive!)}\\ \mathbf{o}_t &= \sigma(W_o[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_o) &&\text{output gate}\\ \mathbf{h}_t &= \mathbf{o}_t \odot \tanh(\mathbf{c}_t) &&\text{hidden state} \end{aligned} \]
\[ \frac{\partial \mathcal{L}}{\partial \mathbf{c}_t} = \frac{\partial \mathcal{L}}{\partial \mathbf{c}_{t+1}} \odot \mathbf{f}_{t+1} + \text{(terms via } \mathbf{h}_t\text{)}. \] When \(\mathbf{f}_{t+1} \approx \mathbf{1}\) the gradient flows back undamped — the "constant error carousel" of Hochreiter & Schmidhuber (1997).
the lighter sibling A GRU does the same job with 4 equations and ~25% fewer parameters — preferred under compute constraints.
The breakthrough idea · attention

What if the model could look at every word at once?

The bottleneck in older translation models: the encoder had to squeeze the whole input into one fixed vector. Bahdanau et al. (2015) let the decoder instead look back at every input word and weight the ones it needs right now.

For decoder state \(\mathbf{s}_t\) and each encoder state \(\mathbf{h}_i\), an alignment score: \[ e_{t,i} = \mathbf{v}^\top \tanh\!\left(W_s\,\mathbf{s}_t + W_h\,\mathbf{h}_i\right). \]
a usable intuition "Attention" is just a learned, soft lookup: given what I'm doing now, which parts of the input should I read most carefully?
Attention · the payoff

Attention weights are a readable map of what the model used

The scores get turned into percentages that sum to 100% — a probability distribution over input positions — and the model reads a weighted blend of those positions.

\[ \alpha_{t,i} = \frac{\exp(e_{t,i})}{\sum_j \exp(e_{t,j})}, \qquad \mathbf{c}_t = \sum_{i} \alpha_{t,i}\,\mathbf{h}_i. \] The \(\{\alpha_{t,i}\}\) form a probability distribution over encoder positions.
the bigger picture A decoder generating a sentence about net income puts high weight \(\alpha_{t,i}\) on exactly the transcript tokens where income figures appear. Attention is interpretable.
the move that follows Apply attention within a single sequence — every word attending to every other word — and you get self-attention, the heart of the Transformer.
02

The Transformer

Self-attention, many heads in parallel, and the encoder block that stacks into every modern LLM.
Self-attention, mechanically

Every word asks every other word: "are you relevant to me?"

Each word produces three roles — a query (what I'm looking for), a key (what I offer), and a value (what I'd contribute). Matching queries to keys decides how much of each value to pull in.

From input \(X \in \mathbb{R}^{n \times d_{\text{model}}}\), project to queries, keys, values: \[ Q = XW^Q,\quad K = XW^K,\quad V = XW^V, \] then \[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V. \] Here \(QK^\top \in \mathbb{R}^{n \times n}\) with entry \((i,j) = \mathbf{q}_i^\top\mathbf{k}_j\); a row-wise softmax gives weights \(A_{ij} > 0,\ \sum_j A_{ij}=1\); token \(i\)'s output is the convex combination \(\sum_j A_{ij}\,\mathbf{v}_j\). (Why the \(\sqrt{d_k}\)? See the appendix.)
the causal mask In a generative (decoder) model we set \(\tilde{S}_{ij} = -\infty\) for \(j>i\), so a token can never "see" words that come after it — otherwise it would cheat at predicting the next word.
Many heads

Run attention several times in parallel — each head a different lens

One attention pattern can only capture one kind of relationship. Multi-head attention runs several in parallel, then concatenates them — so different heads can track grammar, references, and figures at once.

\[ \text{head}_i = \text{Attention}(XW^Q_i,\,XW^K_i,\,XW^V_i), \qquad \text{MultiHead}(X) = \text{Concat}(\text{head}_1,\ldots,\text{head}_h)\,W^O. \] Original Transformer: \(h = 8\), \(d_k = d_v = 64\), \(d_{\text{model}} = 512\).

What do heads learn?

  • Lower layers — syntax: subject–verb agreement, negation scope
  • Deeper layers — semantics: entity co-reference, temporal relations
  • Financial models — heads specialize on "last quarter" → figures, entity linking
the cost of looking everywhere Complexity is \(O(n^2 d_{\text{model}})\) per layer. At \(n = 4{,}096\) tokens that's ≈134M values per layer — which is why FlashAttention (Dao et al., 2022) exists.
Putting it together

One Transformer block: attend, then think, with safety rails

A block does two things — mix information across words (attention), then process each word on its own (a small feed-forward network) — each wrapped in a "shortcut" that keeps training stable.

Two sub-layers with residual connections and Layer Normalisation: \[ Z^{(\ell)} = \text{LayerNorm}\!\left(X^{(\ell)} + \text{MultiHead}(X^{(\ell)})\right), \] \[ X^{(\ell+1)} = \text{LayerNorm}\!\left(Z^{(\ell)} + \text{FFN}(Z^{(\ell)})\right), \] with feed-forward network \[ \text{FFN}(\mathbf{z}) = \max(0,\,\mathbf{z}W_1 + \mathbf{b}_1)\,W_2 + \mathbf{b}_2, \quad d_{\text{ff}} = 4\,d_{\text{model}}. \]
Why the "shortcut" (residual) connections are critical
They give gradients a direct pathway from output back to input, eliminating the multiplicative vanishing-gradient problem across layer depth — the same disease that crippled RNNs, now cured by design.
scale Base Transformer: \(L = 6\) layers. Modern LLMs: \(L = 32\text{–}96\) layers, \(d_{\text{model}} = 4{,}096\text{–}8{,}192\).
03

Pre-training and the LLM landscape

Two ways to learn from raw text — fill-in-the-blank vs. predict-the-next-word — and who ships what.
Recipe 1 · BERT

BERT learns by playing fill-in-the-blank

Hide 15% of the words in a sentence and train the model to guess them from the words on both sides. Reading in both directions makes it a powerful reader — but not a writer.

BERT = Bidirectional Encoder Representations from Transformers (Devlin et al., 2019). Mask a set \(\mathcal{M}\) of tokens and predict the originals: \[ \mathcal{L}_{\text{MLM}}(\theta) = -\sum_{i \in \mathcal{M}} \log p_\theta\!\left(w_i \mid \mathbf{x}_{\setminus \mathcal{M}}\right). \] Masking protocol: 80% replaced with [MASK], 10% with a random token, 10% left unchanged (forcing a representation for every token). Architecture: encoder-only, fully bidirectional attention.
best for Classification (sentiment, credit rating), token-level extraction (NER, metric extraction), and semantic similarity. Not for generation.
Recipe 2 · GPT

GPT learns by always guessing the next word

Read left to right and predict what comes next, over and over. That single skill, at scale, becomes fluent generation — and it's why one architecture can draft, answer, and reason.

\[ \mathcal{L}_{\text{CLM}}(\theta) = -\sum_{t=1}^{T} \log p_\theta(w_t \mid w_1,\ldots,w_{t-1}). \] The causal mask enforces \(j \leq i\) only, giving natural autoregressive generation.

Which architecture for which job?

Earnings-call summarisationEnc–Dec (T5)Seq2seq; full input read

reasoning models (o1, o3, DeepSeek-R1) Trained via RL with verifiable rewards to produce chain-of-thought traces. Same decoder architecture, different training regime. They outperform standard generation on FinQA-type tasks — but do not eliminate hallucination.

04

Sampling and structured generation

The dials that control creativity vs. determinism — and how to force clean, machine-readable output.
The creativity dial

One knob decides whether the model plays it safe or improvises

At each step the model has a probability over possible next words. Temperature, nucleus, and beam search are different rules for picking from that distribution.

  • Temperature — low = predictable, high = adventurous
  • Nucleus (top-p) — only sample from the smallest set of words covering probability p
  • Beam search — keep the B best partial sentences and expand all
Temperature scaling of logits \(z_i\): \[ p_i(\tau) = \frac{\exp(z_i/\tau)}{\sum_j \exp(z_j/\tau)}. \] \(\tau\to 0\): greedy (deterministic); \(\tau = 1\): standard; \(\tau\to\infty\): uniform. Nucleus (top-\(p\)) sampling keeps the smallest set whose mass reaches \(p\): \[ \mathcal{V}_p = \arg\min_S \!\left\{|S| : \sum_{i \in S} p_i(\tau) \geq p\right\}, \] adapting vocabulary size to model confidence; common defaults \(p = 0.9\text{–}0.95\).
beam search trade-off Higher-probability sequences (good for ROUGE/BLEU tasks), but prone to repetitive "degeneration" on open-ended generation.
Choosing the dial in finance

If a machine reads the output, set temperature to zero

The right setting depends entirely on whether you want one correct answer or a range of ideas.

TaskTemperatureSampling
Structured extraction (earnings, covenants)\(\tau = 0\) (greedy)—
Sentiment labelling\(\tau \approx 0.1\)greedy / top-k
Document summarisation\(\tau \approx 0.3\text{–}0.5\)top-p = 0.9
Regulatory Q&A / compliance\(\tau \approx 0.1\)greedy + RAG
Scenario / stress-test narrative\(\tau \approx 0.7\text{–}1.0\)top-p = 0.95
Rule of thumb
If the output feeds a downstream system or is compared to ground truth: \(\tau = 0\). If a human analyst values breadth and novelty: \(\tau \in [0.5, 1.0]\) with nucleus.
self-consistency Run \(K\) samples at \(\tau > 0\) and take the majority vote — especially effective for multi-step numerical reasoning (FinQA-type tasks).
Forcing clean output

How to make a model return data, not prose

To extract figures into a database you need valid JSON every time. Constrained decoding blocks any token that would break the required schema before the model can pick it.

A binary mask over the vocabulary at step \(t\): \[ M_t[i] = \mathbf{1}\!\left[\text{token } i \text{ can extend } x_{<t} \text{ to a string in } \mathcal{L}(\mathcal{G})\right]. \] Invalid tokens receive \(-\infty\) log-probability before softmax, so they can never be sampled. 
class EarningsReport(BaseModel):
    revenue_bn: Optional[float] = None
    net_income_bn: Optional[float] = None
    eps: Optional[float] = None

  tools=[{"name": "record_earnings",
          "input_schema": EarningsReport.model_json_schema()}],
  tool_choice={"type": "tool", "name": "record_earnings"},
  ...
)

Anthropic tool use + Pydantic: the schema defines exactly what fields come back.

Even valid output can be wrong

Three failure modes constrained decoding does NOT fix

Guaranteeing the shape of the answer says nothing about whether the numbers inside it are right.

  1. Numerically wrong-but-valid — a clean number that's simply incorrect (pair with \(\tau=0\) + RAG)
  2. Optional fields — model emits an explicit null (at least detectable)
  3. Unit confusion — millions vs. billions; requires application-level validation
the lesson Schema validity is necessary, not sufficient. Always validate values against the source and expected units.
05

APIs, RAG, and hallucinations

What it costs to run these models, how to ground them in real documents, and where they lie.
What you actually pay for

Models bill by the token — and token math decides your budget

Before processing, text is chopped into sub-word "tokens" (BPE). You pay per token in and per token out, so the same job can cost wildly different amounts across providers.

BPE iteratively merges the most frequent adjacent symbol pair; vocabularies of 32k–100k sub-words. Rule of thumb: 1 English word ≈ 1.3 tokens. "EBITDA" → ["EB","IT","DA"]; rare tickers fragment more. Cost is linear in token counts: \[ \text{cost} = N_{\text{in}} \cdot c_{\text{in}} + N_{\text{out}} \cdot c_{\text{out}}. \]
ModelInput ($/MTok)Output ($/MTok)
GPT-4o$5.00$15.00
Claude 3 Haiku$0.25$1.25
Llama 3 self-hosted≈$0≈$0
worked example 10,000 transcripts × (7,000 input + 400 output tokens): GPT-4o ≈ \(10{,}000 \times (0.035 + 0.006) = \$410\); the same workload on Claude Haiku ≈ $20 (20× cheaper). Prompt caching a shared system-prompt prefix cuts up to 90% of cost.
Grounding the model

RAG: don't trust the model's memory — hand it the document

Retrieval-Augmented Generation fetches the most relevant passages first, then asks the model to answer using only those — so every claim traces back to a real source.

\[ \begin{aligned} &\textbf{1. Retrieval:} && \text{score passages vs. query } q;\ \text{return top-}k\ \text{set } \mathcal{R}(q)\\ &\textbf{2. Augmentation:} && \tilde{q} = [d_{(1)},\ldots,d_{(k)},\,q]\\ &\textbf{3. Generation:} && a \sim P_\theta(\,\cdot \mid \tilde{q}) \end{aligned} \]
why RAG for finance Update the knowledge base by adding new filings — no retraining. Every factual claim is traceable to a retrieved passage (auditability). It's the dominant architecture for production document Q&A.
FinanceBench lesson (Zhang et al., 2024) GPT-4-Turbo with a retrieval system incorrectly answers or refuses 81% of questions from SEC filings. Naive RAG is insufficient for high-stakes financial QA.
The core danger

Hallucination: fluent, confident, and wrong

A hallucination is output that sounds authoritative but isn't supported by the input or the real world. In finance, three kinds do the most damage.

  • numerical D/E ratio of 1.8 reported as 0.8
  • entity wrong company or regulatory body
  • citation a fabricated regulation, ruling, or standard
SelfCheckGPT: draw \(N\) samples; hallucinated claims are inconsistent across samples. FActScore: decompose output into atomic claims; verify each against retrieval. Inner confidence (Chen et al., 2024): trading only on high-confidence LLM signals achieves Sharpe +20% vs. the unconditional benchmark.
root cause Models are trained to produce plausible text. Setting \(\tau = 0\) removes randomness but not hallucination — the most probable completion can still be false.
The defense

No single fix — stack five defenses together

In production you layer complementary techniques; none is sufficient alone.

  1. RAG — ground answers in verified passages; the model rarely fabricates figures absent from retrieved text
  2. Self-consistency sampling — \(K\) chains, majority vote; best for multi-step numerical reasoning
  3. Chain-of-thought — force explicit steps, each independently verifiable
  4. Claim decomposition (FActScore) — break output into atomic claims; flag unsupported ones before review
  5. Confidence + abstention — have the model flag uncertainty; calibrate thresholds against accuracy
reasoning models help, but don't close the gap They reduce hallucination on tasks with verifiable steps (FinQA), but extrinsic hallucination on obscure regulatory facts persists. All five techniques remain necessary.
Using these responsibly

The legal and statistical traps an MBA must know

Beyond accuracy, deploying LLMs in finance runs into regulation, contaminated back-tests, privacy law, and built-in geographic bias.

  • EU AI Act (Reg. 2024/1689, in force Aug 2024) — credit scoring, insurance pricing, AI-assisted securities pricing are high-risk: conformity assessments, human oversight, transparency to regulators
  • Look-ahead bias (Didisheim et al., 2025) — LLMs can reconstruct historical time series from memory, producing spurious back-test predictability; memorised data is contaminated data
  • GDPR — fine-tuning or RAG on client personal data triggers right-to-erasure; a model trained on personal data can't satisfy erasure without retraining
  • Geographic bias — all five major benchmarks are North American / English; US-trained sentiment models may misread Japanese, Chinese, or Indian financial contexts

Appendix — extended derivations & benchmarks

The √dₖ variance proof, positional encoding, financial benchmarks, and retrieval internals. Beyond the core lecture.
Appendix · the missing √dₖ

Why divide attention scores by √dₖ?

Without scaling, dot-product scores grow with dimension, the softmax saturates, and gradients die — reintroducing the very problem attention was meant to solve.

Suppose \(q_\ell, k_\ell \overset{\text{i.i.d.}}{\sim} (0,1)\) for \(\ell = 1,\ldots,d_k\). Step 1. Variance of the dot product: \[ \text{Var}\!\left(\mathbf{q}_i^\top \mathbf{k}_j\right) = \sum_{\ell=1}^{d_k} \text{Var}(q_\ell k_\ell) = d_k. \] Step 2. For large \(d_k\), logits concentrate near \(\pm\sqrt{d_k}\), where the softmax gradient \(\sigma(z)(1-\sigma(z)) \approx 0\). Step 3. Dividing by \(\sqrt{d_k}\) renormalises: \[ \text{Var}\!\left(\frac{\mathbf{q}_i^\top \mathbf{k}_j}{\sqrt{d_k}}\right) = 1. \]
Consequence
Without scaling, the softmax saturates and gradients vanish — the same problem LSTMs were designed to fix, now reappearing inside the attention layer.
Appendix · where is the word?

Self-attention is order-blind — so position is injected

Self-attention treats input as a set: shuffle the words and the math is unchanged. A positional encoding tags each token with its location so order can be recovered.

Self-attention is permutation-equivariant, so position must be added explicitly: \[ \text{PE}_{(\text{pos},2i)} = \sin\!\left(\tfrac{\text{pos}}{10000^{2i/d}}\right), \quad \text{PE}_{(\text{pos},2i+1)} = \cos\!\left(\tfrac{\text{pos}}{10000^{2i/d}}\right). \] Small \(i\): high frequency (fine position); large \(i\): low frequency (coarse position). Rotation property: for any fixed offset \(k\), \[ \begin{pmatrix} \text{PE}_{(\text{pos}+k,2i)} \\ \text{PE}_{(\text{pos}+k,2i+1)} \end{pmatrix} = \begin{pmatrix} \cos\omega_i k & -\sin\omega_i k \\ \sin\omega_i k & \cos\omega_i k \end{pmatrix} \begin{pmatrix} \text{PE}_{(\text{pos},2i)} \\ \text{PE}_{(\text{pos},2i+1)} \end{pmatrix}. \] The rotation depends only on \(k\), not absolute position ⇒ relative positions are linearly decodable.
the modern variant RoPE (Rotary Position Embeddings) bakes the rotation directly into query–key dot products. Used in Llama, Mistral, Qwen.

Appendix · retrieval internals

Dense, sparse, or hybrid: how RAG finds passages

Dense retrieval matches meaning; sparse (BM25) matches exact strings like CUSIPs and tickers; hybrid fuses both. The combination is the production default.

Dense (cosine of bi-encoder embeddings): \[ s_{\text{dense}}(q, d_i) = \frac{\phi(q)\cdot\phi(d_i)}{\|\phi(q)\|\,\|\phi(d_i)\|}, \] with SBERT bi-encoder + FAISS / HNSW for ANN search. BM25 is sparse lexical, strong on exact matches. Hybrid via Reciprocal Rank Fusion: \[ \text{RRF}(d) = \sum_{r \in \{\text{dense},\,\text{sparse}\}} \frac{1}{60 + \text{rank}_r(d)}, \] robust to score-scale differences with no learned weighting parameter.
