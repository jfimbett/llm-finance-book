\section{Text as Financial Data}

\begin{frame}{Text as Financial Data}
  \begin{itemize}
    \item Financial text spans social media, news wires, regulatory filings, earnings transcripts, and central bank communications
    \item Each source carries distinct informational content and noise characteristics
    \item Preprocessing (tokenization, normalization, deduplication) is critical before modeling
    \item Volume and velocity of financial text have grown dramatically, making automated NLP indispensable
  \end{itemize}
\end{frame}

\section{Sentiment Analysis Methods}

\begin{frame}{Sentiment Analysis Methods}
  \begin{itemize}
    \item Lexicon-based methods use curated word lists (e.g., Loughran-McDonald) to score documents without training data
    \item Fine-tuned classifiers like FinBERT and SEC-BERT adapt pre-trained transformers to financial language
    \item LLMs enable zero-shot and few-shot sentiment classification without domain-specific fine-tuning
    \item Each approach involves trade-offs between interpretability, cost, accuracy, and scalability
  \end{itemize}
\end{frame}

\section{Applications by Text Source}

\begin{frame}{Applications by Text Source}
  \begin{itemize}
    \item Social media (Twitter/X, Reddit, StockTwits) reflects retail investor sentiment and can predict short-term price movements
    \item Earnings call transcripts reveal management tone and forward-looking language beyond reported numbers
    \item SEC filings (10-K/10-Q) encode risk factor language and MD\&A readability as signals of firm quality
    \item Central bank communications (FOMC minutes, ECB statements) move asset prices and yield curves
  \end{itemize}
\end{frame}

\section{From Sentiment to Tradeable Signal}

\begin{frame}{From Sentiment to Tradeable Signal}
  \begin{itemize}
    \item Raw sentiment scores must be aggregated, standardized, and tested for stationarity before use in models
    \item Event studies isolate the causal impact of sentiment-laden disclosures on returns around earnings and FOMC meetings
    \item Long-horizon predictability differs from short-horizon: different aggregation windows and decay rates apply
    \item Signal construction requires careful attention to look-ahead bias and publication timing of source documents
  \end{itemize}
\end{frame}

\section{Practical Pipeline}

\begin{frame}{Practical Pipeline}
  \begin{itemize}
    \item Data collection leverages EDGAR full-text search, social media APIs, and commercial news data vendors
    \item Batch inference suits historical backtesting; streaming inference is required for live trading applications
    \item Evaluation relies on inter-annotator agreement metrics (Krippendorff alpha) alongside precision, recall, and F1
    \item A robust pipeline integrates data ingestion, preprocessing, model inference, signal storage, and monitoring
  \end{itemize}
\end{frame}
