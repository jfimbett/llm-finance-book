Recap — the moving parts

The agent loop, the ReAct format, where memory lives, and how we grade RAG.
Recap · how an agent thinks and acts

An agent is a loop: think, act, observe, repeat

Plain version: the model writes a thought, calls a tool, reads what comes back, and updates what it knows — looping until it can answer. The thoughts are the audit trail.

ReAct, in one line
A trajectory alternates Thought → Action → Observation until a Finish. Each thought says why the next tool is called; each observation grounds the next step in real data.
why it matters for finance A model guessing Apple's revenue can be wrong. A model that calls the API returns the exact figure. Actions trade guesses for evidence.
The agent is a triple \((\pi,\mu,\phi)\): a memory update \(\phi:\text{obs}\times\text{mem}\to\text{mem}\), a policy \(\pi:\text{mem}\to\Delta(\text{actions})\) implemented by the LLM, and an environment transition \(\mu:\text{action}\times\text{env}\to\text{obs}\). A ReAct trajectory is \[ \tau = (T_1, A_1, O_1,\; T_2, A_2, O_2,\; \ldots,\; T_n, \textsc{Finish}). \] Each \(T_i\) is a natural-language thought, each \(A_i\) a tool call, each \(O_i\) the tool's return. The grounding principle: the model's internal estimate \(\hat f(t)\neq f(t)\) in general, but calling the API returns the exact \(f(t)\).

RAGAS metrics

  • Faithfulness — answer's claims are backed by retrieved context
  • Answer Relevance — answer addresses the query
  • Context Precision — retrieved chunks are relevant
  • Context Recall — context covers the ground-truth claims
finance priority Faithfulness first. An unfaithful answer is a hallucinated claim, which is a compliance risk. Rule of thumb: faithfulness \(< 0.85\) → block automated publication.

Problem 2 · 10 min

Design a tool schema for a bond calculator

You're building an agent for a fixed-income desk. The tool compute_bond_price returns the present value of a bond from its cash flows and yield, supporting annual and semi-annual coupons.

Fields the schema must include
  • Face value (USD, any positive value)
  • Annual coupon rate (decimal)
  • Years to maturity (positive integer)
  • Yield to maturity (decimal; annual, not semi-annual)
  • Coupon frequency (annual or semi-annual)
for each field Write a Field(description=...) with units, a valid range, and at least one example. Use ge, le, or gt constraints.
The present value of a bond with period coupon \(C\), period yield \(y\), \(n\) periods and face value \(F\) is \[ P = \sum_{t=1}^{n}\frac{C}{(1+y)^{t}} + \frac{F}{(1+y)^{n}}. \] For semi-annual coupons, halve the annual rate and yield and double the number of periods.

Case study · step 1

Chunk the documents, then embed every chunk

Split each call into sentences, then turn each sentence into a vector — a point in space where similar meanings sit close together.

from sentence_transformers import SentenceTransformer
import numpy as np

# --- Sample corpus (use provided strings in the lab notebook) ---
docs = {
    "apple_q4_2023": "Revenue guidance for Q1 2024 is $89-90B ...",
    # ... add remaining documents
}

# --- Chunk each document into sentences ---
def sentence_chunk(text):
    return [s.strip() for s in text.split('.') if len(s.strip()) > 20]

chunks = []
for doc_id, text in docs.items():
    for sent in sentence_chunk(text):
        chunks.append({"doc_id": doc_id, "text": sent})

# --- Embed all chunks ---
model = SentenceTransformer("all-MiniLM-L6-v2")
embeddings = model.encode([c["text"] for c in chunks])

Case study · step 2

Retrieve the closest chunks, then generate a grounded answer

For a query, find the nearest chunks by cosine similarity, paste them into the prompt, and force the model to answer only from that context.

from sklearn.metrics.pairwise import cosine_similarity

def retrieve(query, k=3):
    q_emb = model.encode([query])
    sims = cosine_similarity(q_emb, embeddings)[0]
    top_k = np.argsort(sims)[::-1][:k]
    return [chunks[i] for i in top_k]

def generate_answer(query, context_chunks, llm_fn):
    context = "\n".join(c["text"] for c in context_chunks)
    prompt = f"""Answer based ONLY on the context below.
If the answer is not in the context, say "Not found in context."

Context:
{context}

Question: {query}
Answer:"""
    return llm_fn(prompt)

Discussion — design decisions

When does one agent become two? When does a concern become a hook?
Discussion · 5 min · in pairs

When would you add a second agent — or a hook?

Add a second agent when…
  • the task splits into sub-tasks with different tool needs
  • you need a critic / compliance checker separate from the generator
  • parallelism can cut latency
Add a hook when…
  • a cross-cutting concern (audit, cost) must touch every action
  • the concern is owned by a different team than the agent developer
  • you must enforce a policy without editing the agent's core logic
governance question Your earnings-call agent produces a structured summary that a junior analyst uses to write a client note. What human-in-the-loop pattern would you impose, and why?

Solutions · Problem 2

A reference Pydantic schema for the bond tool

The docstring carries the formula into the model's context; the Field descriptions and constraints shrink the error surface.

from pydantic import BaseModel, Field
from typing import Literal

class BondPriceInput(BaseModel):
    """
    Bond price = sum_{t=1}^{n} C/(1+y)^t + F/(1+y)^n
    where C = coupon payment, y = period yield,
          n = total periods, F = face value.
    """
    face_value: float = Field(
        description="Face value in USD (e.g. 1000.0)", gt=0.0
    )
    annual_coupon_rate: float = Field(
        description="Annual coupon rate as a decimal (e.g. 0.05 for 5%)",
        ge=0.0, le=1.0
    )
the move Treat the schema as a first-class artifact: descriptions and constraints are part of the prompt.

the pattern A Literal with a default turns an open free-text argument into a closed, validated choice — the model can't pass an unsupported convention.
Wrap-up

Five takeaways to carry out of the lab

  1. ReAct grounds reasoning in evidence. Each observation replaces a possibly-hallucinated guess; the thought steps explain why each action is chosen.
  2. Tool schemas are a design artifact. Field descriptions propagate into the model's context; Pydantic constraints reduce the error surface.
  3. RAG evaluation must be decomposed. Faithfulness (generation) and context recall (retrieval) fail for different reasons and need different fixes.
  4. Metadata filtering is not optional. Unconstrained semantic search is noisy for named-entity queries — add document metadata at ingestion time.
  5. Governance matches reversibility. Pre-execution approval for irreversible actions; post-hoc audit for high-volume, low-stakes tasks.

Appendix · stretch problem

Add hybrid search and a compliance hook

Take the dense-retrieval RAG you built and make it both sharper on keywords and safer before it publishes.

Part 1 — hybrid retrieval
  • Add a BM25 sparse scorer alongside the cosine retriever
  • Combine the two scores after min-max normalisation
  • For query 3 ("export controls"), does lowering \(\alpha\) toward \(0.3\) improve which document is retrieved? Explain why.
Part 2 — compliance hook
  • Write a hook \(h(\mathcal{E},\mathcal{M})\) that runs before every Finish and blocks publication when faithfulness \(< 0.85\)
  • What observation does the agent receive when the hook fires, and what recovery action should the policy take?
With min-max-normalised scores \(\widetilde{\text{cosine}}\) and \(\widetilde{\text{BM25}}\), \[ \text{score} = \alpha\,\widetilde{\text{cosine}} + (1-\alpha)\,\widetilde{\text{BM25}}. \] Lowering \(\alpha\) tilts toward exact keyword matching — which helps a rare, literal phrase like "export controls" that dense embeddings may smear into nearby topics.
deliverable The modified retrieve function plus a 3–4 line description of the hook's event, condition, and handler.
