# Lecture 4: LLM Agents and Finance Applications

## Learning Objectives

By the end of this lecture, students should be able to:

1. Describe the perceive–reason–act cycle and distinguish it from single-turn inference.
2. Implement a ReAct-style agent loop with tool use on a financial task.
3. Design and register custom tools for an LLM agent framework.
4. Choose the right memory mechanism (in-context, RAG, long-term) for a given application.
5. Build a retrieval-augmented generation pipeline for financial documents.
6. Orchestrate multi-agent workflows and reason about latency and cost trade-offs.
7. Apply agent systems to earnings analysis, report generation, and filing search.
8. Apply human-in-the-loop governance and defend against prompt injection attacks.
