Recap — the two formulas you'll need

Scaled dot-product attention, and the back-of-envelope cost model for API calls.
Recap · for Problem 1

Attention is a weighted average — the weights are learned

Each token looks at every other token, decides how much each one matters, and blends their value vectors in those proportions. The blending weights are what "attention" computes.

The formula, on the surface
\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) V \] \(Q\) = queries, \(K\) = keys, \(V\) = values, \(d_k\) = head dimension.
Given query matrix \(Q\), key matrix \(K\), value matrix \(V\), head dimension \(d_k\):
  1. Raw scores \(S = QK^\top\) — shape \(n\times n\); entry \((i,j)=\mathbf{q}_i^\top\mathbf{k}_j\).
  2. Scale \(\tilde S = S/\sqrt{d_k}\) — keeps magnitudes from blowing up as \(d_k\) grows.
  3. Softmax each row of \(\tilde S\) to get attention weights \(A\) (each row sums to 1).
  4. Output \(O = AV\) — each row is the attention-weighted sum of value vectors.
Softmax: \(\;\text{softmax}(\mathbf{x})_i = \dfrac{e^{x_i}}{\sum_j e^{x_j}}\). Numerical-stability tip: subtract \(\max(\mathbf{x})\) before exponentiating.
why the \(\sqrt{d_k}\) Without it, larger heads produce larger dot products, the softmax saturates, and gradients vanish. Dividing by \(\sqrt{d_k}\) keeps the scores in a sane range.
Recap · for Problem 2

What an API call costs is just arithmetic on tokens

Providers bill per token, separately for what you send in and what comes back out. Words convert to tokens at a rough fixed rate, so a cost estimate is a one-line calculation.

Rules of thumb
\[ \text{tokens} \approx 1.3 \times \text{words} \] \[ \text{cost} = N_{\text{in}}\cdot c_{\text{in}} + N_{\text{out}}\cdot c_{\text{out}} \] \(N\) = token counts; \(c\) = price per token (quoted per million tokens, MTok).
ModelInput (\(\$\)/MTok)Output (\(\$\)/MTok)
GPT-4o\(\$\)5.00\(\$\)15.00
Claude 3 Haiku\(\$\)0.25\(\$\)1.25
Llama 3 (A100)\(\$\)0 (compute only)\(\$\)0

Self-hosted: an A100 GPU runs about \(\$\)3/hr on cloud spot markets — you pay for time, not tokens.

Problem 1 — attention by hand (15 min)

Run the full attention computation on a tiny financial sequence, by hand, on paper.
Problem 1 · setup

Three tokens, two dimensions, small integers

Our sequence is three tokens of a financial sentence: revenue, increased, significantly. We use head dimension \(d_k=2\) so the arithmetic stays on one page.

Given matrices (row \(i\) = token \(i\))
\[ Q = \begin{pmatrix} 1 & 0 \\ 2 & 1 \\ 0 & 1 \end{pmatrix},\quad K = \begin{pmatrix} 1 & 1 \\ 0 & 2 \\ 1 & 0 \end{pmatrix},\quad V = \begin{pmatrix} 1 & 0 \\ 0 & 1 \\ 1 & 1 \end{pmatrix} \] Row 1 = revenue, row 2 = increased, row 3 = significantly.
why this is fair game \(d_k=2\) is unrealistically small — GPT-3 uses \(d_k=64\), GPT-4 about \(128\). The algebra is identical; we just keep the numbers manageable.
Problem 1 · tasks

Compute it step by step — show your working

  • Task A. Compute \(S = QK^\top\) (a \(3\times 3\) matrix). Recall entry \((i,j)=\mathbf{q}_i^\top\mathbf{k}_j\).
  • Task B. Scale by \(\sqrt{d_k}=\sqrt 2\approx 1.414\); report \(\tilde S = S/\sqrt 2\) to two decimals.
  • Task C. Apply softmax row-wise to \(\tilde S\) to get the attention matrix \(A\) (rows sum to 1); round to three decimals.
  • Task D. Compute the output \(O = AV\) — each row is the attention-weighted sum of value vectors.
Then discuss
  • Which value vector does token revenue attend to most heavily?
  • If \(d_k=64\) instead of 2 with the same raw dot products, would the weights be sharper or flatter?

Case study · the code (1 of 2)

Build the corpus and the index

Embed five passages into vectors, normalise them, and load them into a FAISS inner-product index.

from sentence_transformers import SentenceTransformer
import faiss, numpy as np, anthropic

PASSAGES = [
    "Revenue for FY2023 was $4.2 billion, up 12% year-over-year.",
    "Operating expenses increased to $1.8 billion due to R&D investment.",
    "Net income declined 5% to $820 million, impacted by higher interest expense.",
    "The company holds $2.1 billion in cash and equivalents.",
    "Guidance for FY2024 projects revenue growth of 8-10%.",
]

model = SentenceTransformer("all-MiniLM-L6-v2")
embs = model.encode(PASSAGES, normalize_embeddings=True).astype("float32")
index = faiss.IndexFlatIP(embs.shape[1])
index.add(embs)
Case study · the code (2 of 2)

Retrieve, then generate a grounded answer

Embed the query, pull the top-\(k\) passages, and pass them to Claude with an instruction to answer only from those passages.

def rag_query(query: str, k: int = 2) -> str:
    q_emb = model.encode([query], normalize_embeddings=True).astype("float32")
    _, ids = index.search(q_emb, k)
    context = "\n".join(f"[{i+1}] {PASSAGES[ids[0][i]]}" for i in range(k))
    client = anthropic.Anthropic()
    msg = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=256,
        system="Answer using only the provided passages. Cite passage numbers as [N].",
        messages=[{"role": "user",
                   "content": f"{context}\n\nQuestion: {query}"}],
        temperature=0.0,
    )
    return msg.content[0].text

print(rag_query("What was the net income?"))

Discussion

Which tool fits which job?

For each scenario, identify which approach you would use and why.

  1. Headline triage. A compliance officer must classify 50,000 news headlines/day as material or non-material for insider-trading monitoring. Latency target: under 50 ms per headline. Training data: 10,000 labelled examples.
  2. Live 10-K Q&A. A portfolio manager wants real-time answers about any 10-K in the portfolio, using the most recent filing — documents that were never in any model's training data.
  3. Prospectus drafting. An investment bank must draft the "Risk Factors" section of a new bond prospectus from a structured data dump plus five precedent transactions.
frame your answer around Latency, available labelled data, classification vs generation, need for up-to-date information, cost, and auditability.

Solution · Problem 1 (1 of 2)

Attention by hand — scores and scaling

Tasks A and B: the raw score matrix, then the scaled version.

Task A — \(S = QK^\top\)
\[ S = \begin{pmatrix}1&0\\2&1\\0&1\end{pmatrix}\begin{pmatrix}1&0&1\\1&2&0\end{pmatrix} = \begin{pmatrix}1&0&1\\3&2&2\\1&2&0\end{pmatrix} \]
Task B — \(\tilde S = S/\sqrt 2\)
\[ \tilde S \approx \begin{pmatrix}0.71&0.00&0.71\\2.12&1.41&1.41\\0.71&1.41&0.00\end{pmatrix} \]
Entry \((i,j)=\mathbf{q}_i^\top\mathbf{k}_j\). For example row 2, col 1: \(\mathbf{q}_2=(2,1)\), \(\mathbf{k}_1=(1,1)\), so \(S_{21}=2\cdot 1+1\cdot 1=3\). Dividing every entry by \(\sqrt 2\approx 1.414\) gives \(\tilde S\).
Solution · Problem 1 (2 of 2)

Attention by hand — weights and output

Task C — softmax row-wise → \(A\)
\[ A \approx \begin{pmatrix}0.400&0.200&0.400\\0.561&0.220&0.220\\0.260&0.529&0.211\end{pmatrix} \]
Task D — \(O = AV\)
\[ O \approx \begin{pmatrix}0.800&0.800\\0.781&0.439\\0.740&0.740\end{pmatrix} \]
Row 1 of \(\tilde S\) is \((0.71,\,0,\,0.71)\). Exponentials: \(e^{0.71}+e^{0}+e^{0.71}=2.034+1+2.034=5.068\), giving weights \((0.400,\,0.200,\,0.400)\). Multiplying by \(V\) blends value vectors 1 and 3 most heavily.
reading the answer Token revenue splits its attention evenly between itself and significantly. A larger \(d_k\) with the same dot products makes the softmax sharper — pre-softmax magnitudes are bigger, so the weights concentrate.
Solution · Problem 2 (1 of 2)

API costs — GPT-4o vs Claude 3 Haiku

Token counts per filing
  • Input: \(80{,}000 \times 1.3 = 104{,}000\) tokens of filing text
  • \(+\,1{,}000\) system prompt \(=105{,}000\) input tokens per filing
  • Output: \(600\) tokens per filing
Task A — GPT-4o
\[ 5{,}000 \times\!\left(\tfrac{105{,}000}{10^6}\,5.00 + \tfrac{600}{10^6}\,15.00\right) \] \[ = 5{,}000 \times (0.525 + 0.009) = \$2{,}670 \]
Task B — Claude 3 Haiku
\[ 5{,}000 \times\!\left(\tfrac{105{,}000}{10^6}\,0.25 + \tfrac{600}{10^6}\,1.25\right) \] \[ = 5{,}000 \times (0.02625 + 0.00075) = \$135 \]
the headline number Same workload, same task — Haiku is roughly \(20\times\) cheaper than GPT-4o here (\(\$\)135 vs \(\$\)2,670).
Solution · Problem 2 (2 of 2)

Self-hosting and prompt caching

Task C — self-hosted Llama 3 (A100 at \(\$\)3/hr, 200 filings/hr)
\[ \text{time} = 5{,}000/200 = 25\ \text{hr} \] \[ \text{cost} = 25 \times 3 = \$75 \]
Bonus — prompt caching (90% off the system prompt, Haiku)
\[ 5{,}000\!\left(\tfrac{104{,}000}{10^6}0.25 + \tfrac{1{,}000}{10^6}0.025 + \tfrac{600}{10^6}1.25\right) \] \[ \approx \$133 \]
when to self-host Data-privacy requirements, more than ~\(\$\)3k/yr equivalent spend, or latency that isn't critical. Caching only helps when a large prompt prefix is genuinely fixed across calls.
Wrap-up (1 of 2)

What Problems 1 and 2 actually taught

From Problem 1

  • Attention is a weighted average of value vectors; weights come from softmaxed, scaled query–key dot products
  • Larger \(d_k\) sharpens the softmax — which is exactly why we divide by \(\sqrt{d_k}\)
  • The whole mechanism is differentiable and can be traced by hand

From Problem 2

  • Cost between providers can differ \(20\text{–}35\times\) for the same workload
  • Self-hosting wins when the workload is large and latency is flexible
  • Prompt caching on fixed system prompts yields real savings
Wrap-up (2 of 2)

What the case study taught — and what's next

  • RAG reliably retrieves semantically relevant passages, even from a tiny corpus
  • \(k\) and temperature are meaningful hyperparameters with measurable effects on the output
  • For structured financial extraction: \(\tau=0\), small \(k\), and a grounded system prompt
