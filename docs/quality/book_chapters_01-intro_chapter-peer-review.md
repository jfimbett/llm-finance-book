REFEREE REPORT
==============
Summary: This introductory chapter for "Large Language Models in Finance" opens with a
well-crafted historical narrative tracing textual analysis in finance from dictionary
methods (Harvard IV-4, Loughran-McDonald) through statistical models (LDA, word2vec,
GloVe) to transformers. The chapter provides rigorous formal definitions for classical
text representations (BoW, one-hot encoding, TF-IDF) and word embeddings, accompanied
by a worked numerical example and a case study on PCA visualisation of financial
vocabulary. However, the chapter delivers only three of its eight stated learning
objectives; the remaining five — covering RNNs/LSTMs, the Transformer attention
mechanism, LLM model families, practical API usage, and LLM limitations/regulatory
context — are entirely absent. The chapter as submitted is a high-quality fragment, not
a complete introduction.

Verdict: MAJOR_REVISION

Major Comments:
1. INCOMPLETE SCOPE: The chapter's own Learning Objectives remark box (lines 4-24)
   explicitly commits to eight outcomes. Only LOs 1-3 are addressed by the current
   content. LOs 4 (LSTM and vanishing gradients), 5 (scaled dot-product attention and
   sqrt(d_k) derivation), 6 (generative vs. reasoning LLM families), 7 (API code for
   OpenAI/Anthropic/HuggingFace), and 8 (LLM limitations and regulatory landscape) are
   absent entirely. Publishing a chapter whose stated learning objectives cannot be
   achieved from the chapter's content is a fundamental problem. The author must either:
   (a) complete the missing sections, which is the expected remediation; or (b) revise
   the learning objectives to match the actual scope and relocate the missing content to
   clearly identified subsequent chapters with explicit cross-references.

2. PLACEHOLDER FIGURE: Figure fig:embeddings-pca in Section 4.3 is a TikZ rectangle
   containing the text "[Figure: PCA of financial word embeddings -- generated by
   demo.ipynb]". The surrounding text makes specific empirical claims about observable
   cluster patterns (option/futures/derivative tight cluster, default/credit co-cluster,
   rates cluster cohesion). These claims cannot be verified from the placeholder. The
   figure must be generated and embedded before this section can be assessed as complete.
   The notebook `code/notebooks/01-intro/demo.ipynb` is referenced; the author should
   confirm it exists and produces the claimed output.

3. INCORRECT CITATION (liu2018): The citation on line 1147 for "Bloomberg Financial
   Word2Vec" is acknowledged in a comment to be wrong: "[CITE: Liu et al. (2018).
   Efficient low-rank multimodal fusion... ACL — or appropriate FinNLP reference]". The
   tagged reference `\cite{liu2018}` points to an unrelated paper on multimodal fusion.
   The correct reference for a Bloomberg/financial-domain Word2Vec model must be
   identified and substituted. Citing the wrong paper for an empirical claim is a
   factual error.

4. MISSING PRACTICAL API SECTION (LO 7): For a book targeting mixed academic and
   industry readers (researchers, engineers, quants), the absence of any working code
   demonstrating LLM API usage is a significant gap. Learning Objective 7 specifically
   promises code for OpenAI, Anthropic, and HuggingFace APIs with token cost and latency
   analysis. This is the content most immediately useful to practitioners. The chapter
   should include a hands-on deepdive section with minimal but complete Python examples,
   referencing the companion notebook for fuller treatment.

5. MISSING LIMITATIONS AND REGULATORY SECTION (LO 8): The financial applications of
   LLMs carry specific risks — hallucination in numerical reasoning, data contamination
   in backtests (Didisheim et al. 2025), model risk under supervisory guidance (SR 11-7
   in the US, EBA guidelines in Europe). The "Looking Ahead" section mentions
   hallucinations and memorisation in two sentences. For a chapter that serves as the
   reader's conceptual foundation, a substantive discussion of limitations and the
   evolving regulatory landscape is essential, not optional.

Minor Comments:
1. BROKEN CROSS-REFERENCE: Line 1302, within the caption of fig:ch01-illustration,
   contains `\ref{sec:intro-classical}`. The actual label for the classical
   representations section is `\label{sec:classical}` (line 476). This will render as
   "??" in the compiled PDF. Fix by changing the reference to `\ref{sec:classical}`.

2. FINBERT CITATION AMBIGUITY: Line 218 cites `araci2019finbert` for FinBERT. The name
   "FinBERT" was independently used by Yang et al. (2020, arXiv:2006.08097) for a
   different model. Both papers should be cited at the FinBERT mention with a brief
   footnote clarifying the two distinct models sharing the same name.

3. COSINE SIMILARITY DERIVATION (lines 798-803): The simplification that collapses the
   product of two norms into a single squared norm is valid only because both norms
   happen to be equal (both documents have the same non-zero TF-IDF weights). This
   algebraic coincidence is not noted, which may confuse readers who attempt to
   generalise the computation. Add a clarifying sentence.

4. LOOKING AHEAD SECTION: The current "Looking Ahead" section (Section 5) does not
   provide specific chapter-section pointers for where LOs 4-8 will be covered. It
   should map each of the unaddressed learning objectives to a specific chapter and
   section number, giving the reader a genuine road map rather than a paragraph of
   general forward-looking statements.

5. NOTE ON EXERCISES: Only a single exercise appears (the TF-IDF illustration exercise
   on lines 1306-1316). The chapter's stated scope — if LOs 4-8 were present — would
   warrant additional exercises at beginner [B], intermediate [I], and advanced [A]
   levels covering embeddings (e.g., implement Skip-Gram gradient update), attention
   (e.g., compute attention weights by hand), and API usage. Until those sections are
   written, the exercise set is necessarily incomplete.
