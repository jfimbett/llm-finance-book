\section{The Information Extraction Problem in Finance}

\begin{frame}{The Information Extraction Problem in Finance}
  \begin{itemize}
    \item Financial documents range from highly structured (XBRL filings) to fully unstructured (news, analyst notes)
    \item Key entities of interest: company names, dates, monetary figures, legal covenants, and relationships
    \item Information extraction (IE) transforms raw text into structured, machine-readable representations
    \item Summarization condenses document content while preserving key facts and meaning
  \end{itemize}
\end{frame}

\section{Named Entity Recognition in Finance}

\begin{frame}{Named Entity Recognition in Finance}
  \begin{itemize}
    \item Financial NER must identify domain-specific entity types beyond standard PER/ORG/LOC
    \item Benchmark datasets include FINER-139 (139 fine-grained types), FinNER, and FinRE for relation extraction
    \item Transformer models (BERT, FinBERT) can be fine-tuned on annotated financial corpora for high-accuracy NER
    \item Relation extraction links entities: ownership structures, employment roles, and financial transactions
  \end{itemize}
\end{frame}

\section{Financial Document Summarization}

\begin{frame}{Financial Document Summarization}
  \begin{itemize}
    \item Extractive summarization selects key sentences; abstractive summarization generates novel text
    \item Earnings reports and 10-K filings are primary targets due to their length and standardized structure
    \item Analyst notes and research reports require condensation while preserving investment thesis and risk factors
    \item Long documents (transcripts, S-1 filings) exceed typical context windows, requiring hierarchical or chunked approaches
  \end{itemize}
\end{frame}

\section{Table and Numerical Data Extraction}

\begin{frame}{Table and Numerical Data Extraction}
  \begin{itemize}
    \item SEC XBRL tags provide machine-readable structured data directly from regulatory filings
    \item LLMs can parse and interpret financial tables, inferring column semantics and cross-table relationships
    \item Consistency checking verifies that numerical claims in prose match reported figures in tables
    \item Extraction pipelines must handle diverse table layouts: income statements, balance sheets, footnotes
  \end{itemize}
\end{frame}

\section{Evaluation and Benchmarks}

\begin{frame}{Evaluation and Benchmarks}
  \begin{itemize}
    \item ROUGE measures n-gram overlap; BERTScore captures semantic similarity via contextual embeddings
    \item Factual consistency metrics detect hallucinations — critical in financial applications with legal consequences
    \item Human evaluation by domain professionals assesses relevance, accuracy, and completeness of extractions
    \item Key datasets: ECTSum (earnings calls), EDGAR-Corpus (SEC filings), FiNER, and FinSBD (sentence boundary detection)
  \end{itemize}
\end{frame}
