## Learning Objectives

By the end of this lecture, you should be able to:

1. Articulate the principal privacy risks that arise when financial institutions use large language models, and map those risks to specific attack vectors.
2. Summarise the key provisions of GDPR, CCPA, MiFID II, and DORA that govern AI-assisted processing of personal and financial data.
3. Evaluate the trade-offs between cloud-hosted and locally deployed LLMs for privacy-sensitive financial workflows.
4. Design a text anonymisation pipeline using named entity recognition to detect and mask personal names, addresses, and financial identifiers.
5. Explain differential privacy and federated learning and describe how each technique limits private information leakage during model training.
6. Construct a privacy-aware retrieval-augmented generation pipeline with audit logging and access controls.
