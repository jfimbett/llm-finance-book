\chapter{LLMs and Privacy: Local Deployments and Text De-identification}
\label{ch:privacy-local-models}

\begin{remark}[Learning Objectives]
After completing this chapter, the reader should be able to:
\begin{enumerate}
  \item Articulate the principal privacy risks that arise when financial institutions
        use large language models, and map those risks to specific attack vectors.
  \item Summarise the key provisions of GDPR, CCPA, MiFID II, and DORA that govern
        AI-assisted processing of personal and financial data.
  \item Evaluate the trade-offs between cloud-hosted and locally deployed LLMs for
        privacy-sensitive financial workflows.
  \item Select and deploy an open-weight model (Llama, Mistral, or Phi) for a given
        hardware budget and latency constraint.
  \item Design a text anonymisation pipeline using named entity recognition to detect
        and mask personal names, addresses, and financial identifiers.
  \item Explain differential privacy and federated learning and describe how each
        technique limits private information leakage during model training.
  \item Construct a privacy-aware retrieval-augmented generation pipeline with audit
        logging and access controls.
  \item Apply a structured compliance checklist to assess a planned LLM deployment
        against current regulatory requirements.
\end{enumerate}
\end{remark}
