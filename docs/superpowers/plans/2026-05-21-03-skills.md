```bash
#!/bin/bash
# validate-skills.sh — checks all 22 skill files exist and have required sections
set -euo pipefail

PASS=0; FAIL=0

check_skill() {
  local name="$1"
  local file=".claude/skills/${name}.md"
  if [ ! -f "$file" ]; then
    echo "MISS  $file"
    FAIL=$((FAIL+1))
    return
  fi
  local ok=1
  for section in "## Purpose" "## When to Invoke" "## Inputs Required" "## Steps" "## Expected Output" "## Error Handling"; do
    if ! grep -q "$section" "$file"; then
      echo "NO_SECTION  ${name}: ${section}"
      FAIL=$((FAIL+1))
      ok=0
    fi
  done
  if [ "$ok" -eq 1 ]; then
    echo "  OK  ${name}"
    PASS=$((PASS+1))
  fi
}

# /new-topic

## Purpose
Scaffold a new numbered chapter/lecture/notebook unit with all required directories and placeholder files.

## When to Invoke
When the user wants to add a new topic to the course and the corresponding directories do not yet exist under `book/chapters/`, `course/lectures/`, or `code/notebooks/`.

## Inputs Required
- Topic number as a zero-padded two-digit string (e.g., `"02"`)
- Topic short name as a lowercase hyphenated string (e.g., `"linear-algebra"`)
- `TOPIC.md` present at the project root (used to read project-level defaults)

## Steps

1. If the user has not provided the topic number and short name, ask: "What is the topic number (e.g. 02) and short name (e.g. linear-algebra)?" Wait for the answer before continuing.
2. Construct the directory slug `NN-name` (e.g., `02-linear-algebra`).
3. Check whether `book/chapters/NN-name/` already exists. If it does, stop and ask: "Directory `book/chapters/NN-name/` already exists. Overwrite its contents, or pick a different number?" Do not proceed until the user confirms.
4. Create `book/chapters/NN-name/chapter.tex` by copying `book/chapters/01-intro/chapter.tex` and replacing every occurrence of `"Introduction"` with the title-cased topic name (e.g., `"Linear Algebra"`).
5. Create `book/chapters/NN-name/figures/.gitkeep` (empty file) so git tracks the figures directory.
6. Create `course/lectures/NN-name/notes.md` by copying `course/lectures/01-intro/notes.md` and replacing topic-specific strings (title, number) with the new topic values.
7. Create `course/lectures/NN-name/slides.tex` by copying `course/lectures/01-intro/slides.tex` and applying the same substitutions.
8. Create `course/lectures/NN-name/exercises.md` by copying `course/lectures/01-intro/exercises.md` with topic substitutions.
9. Create `course/lectures/NN-name/solutions.md` by copying `course/lectures/01-intro/solutions.md` with topic substitutions.
10. Create `code/notebooks/NN-name/demo.ipynb` by copying `code/notebooks/01-intro/demo.ipynb` and updating the notebook title cell.
11. Create `code/notebooks/NN-name/exercises.ipynb` by copying `code/notebooks/01-intro/exercises.ipynb` and updating the notebook title cell.
12. Open `docs/STATUS.md`, locate the status table, and append a new row for the topic with `"No"` in every status column (Drafted, Scored, Clarity, Rigor, Completeness, Pedagogy, Style, Released).
13. Run `bash scripts/validate-scaffold.sh` to confirm all expected pairs exist. If validation fails, print the failure lines and stop.
14. Stage and commit:
    ```bash
    git add book/chapters/NN-name course/lectures/NN-name code/notebooks/NN-name docs/STATUS.md
    git commit -m "chore: scaffold topic NN-name"
    ```

## Expected Output
- Three new directories: `book/chapters/NN-name/`, `course/lectures/NN-name/`, `code/notebooks/NN-name/`
- `docs/STATUS.md` updated with a new row for the topic (all statuses `No`)
- A clean git commit containing the new scaffold files

## Error Handling
- **Directory already exists:** Ask the user whether to overwrite or choose a different number. Never overwrite silently.
- **Template files missing:** If `book/chapters/01-intro/chapter.tex` (or any other intro template) does not exist, print "Template file not found: [path]. Run /new-topic for topic 01 first, or check your scaffold." and abort.
- **validate-scaffold.sh fails:** Print the failure lines verbatim. Do not commit. Ask the user to inspect and fix before retrying.
- **TOPIC.md missing:** Continue with scaffolding, but warn: "TOPIC.md not found — run /interview-me to configure the project."
```

# /draft-chapter

## Purpose
Generate a first draft of a book chapter as a complete LaTeX file with sections, theorems, proofs, examples, and bibliography entries.

## When to Invoke
When a chapter scaffold exists (i.e., `book/chapters/NN-name/chapter.tex` is present) but the file still contains placeholder content rather than real chapter material.

## Inputs Required
- Chapter number and short name (e.g., `"02-linear-algebra"`)
- `TOPIC.md` at the project root (subject, audience, depth, quality_threshold)
- Any outline, notes, or reference list the user wants to incorporate (optional but encouraged)

## Steps

1. Read `TOPIC.md` to determine the subject, target audience, depth setting, and `quality_threshold`.
2. Check that `book/chapters/NN-name/chapter.tex` exists. If it does not, tell the user: "Chapter scaffold not found. Run `/new-topic` first." and stop.
3. Read the current content of `book/chapters/NN-name/chapter.tex` to determine whether it is still a placeholder. A placeholder is any file whose main body consists only of the template comment lines and no real section content.
4. Invoke the **literature-reviewer** agent with the chapter topic and audience context. The agent will:
   - Identify 5–10 key sources relevant to the topic
   - Return BibTeX entries for each source
   - Append those entries to `book/bibliography.bib` (deduplicated by key)
5. Invoke the **book-writer** agent with: the chapter topic, audience, depth, any user-provided outline, and the BibTeX keys just added. The agent will produce a complete LaTeX chapter draft including:
   - `\section`, `\subsection` headings
   - Theorem / Lemma / Proof environments as appropriate
   - Worked examples with `\begin{example}...\end{example}`
   - Inline `\cite{}` references to the bibliography entries
6. Invoke the **math-checker** agent on the draft produced in step 5. The agent will verify every derivation, equation, and proof. For each finding rated `FAIL`:
   - Return the corrected derivation to the book-writer agent and ask it to fix that specific location
   - Re-run math-checker on the corrected section
   - Repeat until math-checker returns no `FAIL` findings for this chapter
7. Invoke the **editor** agent on the math-approved draft. The agent will improve clarity, flow, transitions, and consistency of notation.
8. Write the final content to `book/chapters/NN-name/chapter.tex`, replacing the placeholder.
9. Run `/score-content book/chapters/NN-name/chapter.tex` and display the score table.
10. If any dimension score is below `quality_threshold`, run `/refine-until-threshold book/chapters/NN-name/chapter.tex`.
11. Stage and commit:
    ```bash
    git add book/chapters/NN-name/chapter.tex book/bibliography.bib
    git commit -m "feat(chNN): draft [topic name] chapter"
    ```

## Expected Output
- `book/chapters/NN-name/chapter.tex` — complete LaTeX chapter with all sections, theorems, proofs, examples, and citations
- `book/bibliography.bib` — updated with new BibTeX entries for the chapter
- Quality scores at or above `quality_threshold` on all 5 dimensions
- A clean git commit

## Error Handling
- **math-checker returns FAIL after 3 iterations on the same location:** Stop iterating and print: "Math-checker still failing at [location] after 3 passes — human review required." Commit the best available draft with message `"wip(chNN): draft [topic name] — math issues flagged"`.
- **literature-reviewer returns no sources:** Continue without bibliography entries; add a comment `% TODO: add bibliography entries` at the top of the chapter.
- **book/bibliography.bib missing:** Create it as an empty file before appending, and add it to the commit.
- **Chapter already has real content (not a placeholder):** Warn the user: "chapter.tex already has content. Overwrite? (yes/no)" and wait for confirmation.
```

# /draft-lecture

## Purpose
Generate lecture notes and slides derived from an existing book chapter, tailored for classroom delivery.

## When to Invoke
After `/draft-chapter` has been run for the corresponding chapter and `book/chapters/NN-name/chapter.tex` contains real content (not a placeholder).

## Inputs Required
- Chapter/lecture number and short name (e.g., `"02-linear-algebra"`)
- `book/chapters/NN-name/chapter.tex` — must contain real content, not a placeholder

## Steps

1. Read `TOPIC.md` to confirm coupling style (`"loosely coupled"` or `"tightly coupled"`) and audience.
2. Read `book/chapters/NN-name/chapter.tex`. If the file still contains only placeholder content, stop and print: "chapter.tex for NN-name is still a placeholder. Run `/draft-chapter` first." Do not proceed.
3. Read the existing `course/lectures/NN-name/notes.md` and `course/lectures/NN-name/slides.tex` to determine what (if any) content is already present.
4. Invoke the **lecture-writer** agent with the chapter content, the coupling setting, and the target audience. The agent will produce:
   - `notes.md` content: learning objectives, key concepts, narrative explanations, links to chapter sections
   - `slides.tex` content: Beamer frames for each major concept, one idea per frame, with `\begin{itemize}` bullet points, equations, and figure references
5. Invoke the **pedagogy-reviewer** agent on the `notes.md` draft. The agent checks:
   - Whether each stated learning objective is addressed by at least one slide or exercise
   - Whether difficulty progression is appropriate for the target audience
   - Whether scaffolding is present (easy examples before hard ones)
   For each finding rated `BLOCKER`:
   - Return the issue to the lecture-writer agent with instruction to fix that specific gap
   - Re-run pedagogy-reviewer on the revised section
   - Repeat until no `BLOCKER` findings remain
6. Invoke the **editor** agent on `notes.md` to improve clarity and prose flow.
7. Write the final content to `course/lectures/NN-name/notes.md`.
8. Write the final `slides.tex` content to `course/lectures/NN-name/slides.tex`.
9. Run `/score-content course/lectures/NN-name/notes.md` and display the score table.
10. Stage and commit:
    ```bash
    git add course/lectures/NN-name/notes.md course/lectures/NN-name/slides.tex
    git commit -m "feat(lecNN): draft [topic name] lecture"
    ```

## Expected Output
- `course/lectures/NN-name/notes.md` — complete lecture notes with learning objectives, key concepts, and explanations
- `course/lectures/NN-name/slides.tex` — complete Beamer slide deck with one idea per frame

## Error Handling
- **chapter.tex is still a placeholder:** Stop immediately. Print the message in Step 2 above.
- **pedagogy-reviewer returns BLOCKER after 3 iterations:** Stop and print: "Pedagogy issues unresolved after 3 passes at [section] — human review required." Commit the best draft with `"wip(lecNN): draft [topic name] — pedagogy issues flagged"`.
- **notes.md or slides.tex already has real content:** Warn the user and ask for confirmation before overwriting.
- **TOPIC.md missing coupling setting:** Default to `"loosely coupled"` (lecture is a summary of the chapter, not a 1-to-1 mirror).
```

- **exercise-designer produces fewer than 3 exercises:** Ask the agent to produce additional exercises at the missing difficulty level(s) before proceeding to math-checker.

# /draft-figures

## Purpose
Generate TikZ or matplotlib figures for a chapter section and embed them in the LaTeX source.

## When to Invoke
When a chapter has a concept (geometric, algorithmic, data-driven) that would benefit from a diagram or plot and no figure currently exists for it.

## Inputs Required
- Target chapter number and short name (e.g., `"02-linear-algebra"`)
- Section title or description of the concept to illustrate
- Figure type: `"conceptual"` (use TikZ) or `"data plot"` (use matplotlib / Python)

## Steps

1. Read the target section from `book/chapters/NN-name/chapter.tex` to understand the concept being illustrated.
2. Invoke the **figure-designer** agent with the section content, figure type, and any additional context the user provided. The agent will produce a self-contained figure implementation.
3. **If figure type is `"conceptual"` (TikZ):**
   a. The agent produces a `\begin{figure}...\end{figure}` block containing a `tikzpicture` environment.
   b. Insert the block into `book/chapters/NN-name/chapter.tex` immediately after the paragraph that first introduces the concept.
   c. Add `\usepackage{tikz}` to `book/preamble.tex` if not already present.
4. **If figure type is `"data plot"` (matplotlib):**
   a. The agent produces a Python script that saves its output to `book/chapters/NN-name/figures/figure_name.pdf`.
   b. Save the script to `code/notebooks/NN-name/figures/figure_name.py`.
   c. Insert a `\begin{figure}...\end{figure}` block in the appropriate location in `chapter.tex` using `\includegraphics{figures/figure_name}`.
5. Add a descriptive `\caption{...}` and a `\label{fig:NN-figurename}` to the figure block.
6. Run `/build-book` to verify the chapter compiles without errors.
7. Stage and commit:
    ```bash
    git add book/chapters/NN-name/chapter.tex book/preamble.tex
    # If matplotlib figure:
    git add code/notebooks/NN-name/figures/
    git commit -m "feat(chNN): add [description] figure"
    ```

## Expected Output
- A compilable figure embedded in the correct location in `chapter.tex`
- TikZ figures: self-contained in the `.tex` file
- Matplotlib figures: Python source in `code/notebooks/NN-name/figures/` and `\includegraphics` in `chapter.tex`
- `book/main.pdf` updated to include the new figure

## Error Handling
- **TikZ fails to compile:** Print the relevant lines from `main.log`. Ask the figure-designer agent to simplify the TikZ code or switch to a matplotlib equivalent. If simplification still fails, switch to matplotlib.
- **matplotlib script errors:** Print the Python traceback. Return it to the figure-designer agent and ask for a corrected script.
- **pdflatex not found:** Print "Install TeX Live or MiKTeX before running /draft-figures" and abort.
- **Chapter section not found:** Print: "Section '[description]' not found in chapter.tex. Check the section title and try again."
```

# /draft-code

## Purpose
Generate companion code (reusable library functions and notebook cells) for a lecture or chapter.

After lecture notes exist for the topic (i.e., `course/lectures/NN-name/notes.md` contains real content) or a chapter draft exists.

- Programming language from `TOPIC.md` (`python`, `r`, or `julia`)

## Steps

1. Read `TOPIC.md` for the programming language setting. If `language: none`, print: "TOPIC.md specifies no companion code for this project." and stop.
2. Read `course/lectures/NN-name/notes.md` (preferred) or `book/chapters/NN-name/chapter.tex`. If both are placeholders, stop and print: "No real content found for NN-name. Run `/draft-lecture` or `/draft-chapter` first."
3. Invoke the **code-writer** agent with the notes/chapter content and language setting. The agent will produce:

   - Notebook cells for `code/notebooks/NN-name/demo.ipynb` that import from `code/src/` and demonstrate each function
4. Write the module file(s) to `code/src/NN_name/`.
5. Write the notebook cells to `code/notebooks/NN-name/demo.ipynb`.
6. Invoke the **code-reviewer** agent on all files written in steps 4–5. The agent will check for:
   - Correctness bugs (`BUG`)
   - Style or maintainability warnings (`WARNING`)
   - Missing docstrings or type hints (`INFO`)
   For each finding rated `BUG`:
   - Fix the specific location in the source file
   - Re-run code-reviewer on that file
   - Repeat until no `BUG` findings remain
   For `WARNING` findings: fix before committing.
   For `INFO` findings: fix if trivial, otherwise leave a `# TODO` comment.
7. Stage and commit:
    ```bash
    git add code/src/NN_name/ code/notebooks/NN-name/demo.ipynb
    git commit -m "feat(code-NN): add companion code for [topic]"
    ```

## Expected Output
- `code/src/NN_name/` — one or more module files with documented, tested functions
- `code/notebooks/NN-name/demo.ipynb` — filled demo notebook importing from `code/src/`
- No `BUG` or `WARNING` findings from code-reviewer

## Error Handling
- **TOPIC.md language is `none`:** Stop with the message in Step 1.
- **code-reviewer finds BUG after 3 iterations:** Add `# BUG: [description] — needs human review` at the failing location. Commit the partial code with `"wip(code-NN): companion code — bugs flagged"`.
- **code/src/ directory does not exist:** Create it with `mkdir -p code/src/NN_name` before writing.
- **Notebook format invalid:** If the code-writer produces malformed JSON for the `.ipynb` file, print the error and ask the agent to produce the notebook cells as a Python script to `demo.py` instead.

# /refine-until-threshold

## Purpose
Iteratively improve a file until all 5 quality dimension scores reach the project threshold, using the appropriate specialist agent for each failing dimension.

## When to Invoke
Automatically triggered by `gate-check.sh` when a file fails scoring. Can also be invoked manually on any content file at any time.

## Inputs Required
- File path to refine (e.g., `book/chapters/02-linear-algebra/chapter.tex`)
- Optional: `max_iterations` override (integer). If not provided, reads from `TOPIC.md`.

## Steps

1. Read `TOPIC.md`:
   - `quality_threshold` (default `7` if missing)
   - `max_refine_iterations` (default `5` if missing)
   If the caller passed a `max_iterations` override, use that value instead of `TOPIC.md`.

2. Run `/score-content [file_path]` to get the current dimension scores. Store them as the "current scores" for this iteration.

3. Check: are all 5 dimension scores >= `quality_threshold`?
   - **Yes:** Print `"PASS — all dimensions meet threshold (threshold: [N]):"` followed by the score table. Exit with status 0.
   - **No:** Proceed to step 4.

4. Sort the failing dimensions by score (lowest first). Select the dimension with the lowest score as the **target dimension** for this iteration.

5. Map the target dimension to the correct agent using this exact table:

   | Dimension    | Primary Agent      | Secondary Agent (if primary fails 2× on same dimension) |
   |--------------|--------------------|----------------------------------------------------------|
   | clarity      | editor             | humanizer                                                |
   | rigor        | math-checker       | book-writer (to rewrite the unclear section)             |
   | completeness | structure-reviewer | book-writer (to fill identified gaps)                    |
   | pedagogy     | pedagogy-reviewer  | lecture-writer (for lecture files) / book-writer (for chapters) |
   | style        | humanizer          | proofreader                                              |

   Invoke the **primary agent** for the target dimension. Pass it:
   - The full file content
   - The current score for the target dimension (e.g., "Current clarity score: 5.2 / 10")
   - Explicit instruction: "Improve the [dimension] dimension. Focus only on [dimension]-related issues. Do not change mathematical content."

6. Receive the improved content from the agent. Write it back to the file at `[file_path]`, replacing the previous content.

7. Increment the iteration counter.

8. Check: has the same dimension failed in the **2 immediately preceding iterations** (i.e., the primary agent has tried twice without improvement)?
   - **Yes:** Switch to the **secondary agent** for this dimension (see the table in step 5). Invoke it with the same instructions. If the secondary agent also fails to improve the score past threshold, move on to the next-lowest-scoring dimension in the next iteration. Log: "Dimension [name] stuck after 2 passes — switching to next-lowest dimension."
   - **No:** Continue to step 9.

9. Check: is `iteration_counter >= max_refine_iterations`?
   - **Yes:** Print:
     ```
     MAX ITERATIONS REACHED ([N] iterations).
     Remaining failing dimensions: [list with current scores]
     Human review required for: [file_path]
     ```
     Exit with status 1.
   - **No:** Go back to step 2.

## Expected Output
- File at `[file_path]` improved so that all 5 dimension scores >= `quality_threshold`
- Final score table printed showing all dimensions at PASS
- Or: a clear `MAX ITERATIONS REACHED` message with the list of still-failing dimensions and exit status 1

## Error Handling
- **File does not exist:** Print "File not found: [file_path]" and exit with status 1.
- **`/score-content` fails:** Print the error from score-content and exit with status 1. Do not attempt to refine a file that cannot be scored.
- **Same dimension fails for 2 consecutive iterations with the primary agent:** Switch to the secondary agent (per the table in step 5). Log the switch.
- **All remaining failing dimensions have been tried with both primary and secondary agents without reaching threshold:** Print `"MAX ITERATIONS REACHED"` even if `max_refine_iterations` has not been hit, and exit with status 1.
- **`TOPIC.md` missing:** Use defaults: `quality_threshold=7`, `max_refine_iterations=5`. Print a notice.
```

- Optional: scope (e.g., `ch02`, `lec03`) — inferred from changed file paths if not provided

2. If there are no changes (empty output from step 1), print: "Nothing to commit — working tree is clean." and exit.

   - Files in `book/chapters/` with `.tex` extension → `feat`

   - Mixed changes → use the type that applies to the majority of changed files
4. Infer the scope from changed file paths:
   - Path contains `chapters/NN-name` → scope is `chNN` (e.g., `ch02`)
   - Path contains `lectures/NN-name` → scope is `lecNN` (e.g., `lec02`)

   ```
